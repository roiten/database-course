1.1* #20
Создайте и заполните базу данных для хранения статистики прохождения курсов. При желании разрешается дополнять сущности атрибутами. Список сущностей:
* Course - базовая сущность для курса, курсом является квиз (quiz), видео (video) или аудио (audio)
* Квиз состоит из вопросов (QuizQuestion)
* Вопросы могут быть с множественным выбором из вариантов ответа (MultipleChoiceQuestionAvailableValues), либо вопросы на выстраивание последовательности (SequenceQuestionAvailableValues)
  
  

* Каждый квиз оценивается (оценка 2, 3, 4 или 5) по-своему согласно данным из таблицы QuizMark и баллам прохождения (от 0 до 100)
* В системе есть пользователи (User), которых можно назначить на курс через сущность Enrollment
* В сущности User есть статус: активен, неактивен, уволен. Также у пользователя может не быть имени, а только email.
* В сущности Quiz есть статус: загружен, обработан, готов к использованию
* На каждое назначение может быть несколько попыток прохождения материала Attempt
* Для курса типа “квиз” у каждого прохождения сохраняются ответы на вопросы в QuizAttemptAnswer *


* нюанс заключается в том, что нужно придумать, как хранить ответ на последовательность вопросов

1.2* #X (max 50)
Напишите запросы на извлечение данных, используя соединение таблиц и агрегацию. 
Каждый запрос должен выполняться оптимально, для этого добавьте в таблицы соответствующие индексы. Индексы не должны быть избыточными.


Бонус за проверку скорости запросов на больших данных + 2 балла на каждый запрос
Следует заполнить таблицы большим количеством данных (сделать это можно с помощью скрипта). 




Каждый запрос оценивается в 3 балла.
1. * Извлечь имена всех активных пользователей, которые правильно ответили на все вопросы готового к использованию квиза с названием <ваше название>. Квиз должен быть не удалённым курсом. Если у пользователя нет имени, отображать его email. 
2. * Извлечь все прохождения, не пройденные до конца (когда один или несколько вопросов не пройдены). Нужно извлечь имена пользователей и курсов, в которых есть такая ситуация. Пара пользователь - курс не должна дублироваться.
3. * Подсчитать среднее количество вопросов в квизе, на которые правильно ответили уволенные пользователи в период до 2025, а также посчитать посчитать среднее количество вопросов в квизе, на которые правильно ответили активные пользователи в период за 2025. И сделать текстовый вывод, кто проходит квизы лучше. Сделать в единый SQL запрос
4. Посчитать статистику: на какие типы вопросов у готовых квизов и активных пользователей в процентном соотношении больше правильных ответов. Например, на вопросы с последовательностью 50% правильных ответов, а на вопросы с множественным выбором 60% правильных ответов. Проценты сравнить.
5. Среди попыток прохождения пользователя с email <ваш email> квиза <название квиза> извлечь вопросы, на которые пользователь чаще отвечал правильно.
6. Запросом заполнить поле score для отношения Attempt из % правильных ответов среди всех вопросов. Заполнить нужно только для неактивных пользователей у контента, готового к использованию. Дедлайн прохождения курса должен быть не позже марта 2025 года включительно.
7. Извлечь статистику по прохождению квизов, статистика включает: имя пользователя, имя курса, дата лучшей попытки (помимо наивысшего балла, попытка должна быть выполнена в срок диапазона прохождения в Enrollment), значение score лучшей попытки, количество попыток (рассчитывается из ответов на вопросы, а не из отношения Attempt), оценку за тест из отношения QuizMark.
8. Извлечь 10 активных пользователей, которые суммарно набрали наибольшее количество баллов в вопросах с последовательностью в готовых к использованию квизах с максимально допустимой продолжительностью прохождения до 10 минут.
9. Посчитать среднюю оценку (QuizMark) за квизы, если бы там были только вопросы с последовательностью, без вопросов с множественным выбором.
10. Посчитать одну цифру: на какую оценку обучена вся компания среди активных пользователей и готовых к использованию контентов. Считать по всем вопросам вручную для того, чтобы проверить, что попытка пройдена полностью, а не частично.
* Для студентов Института iSpring группы ПС обязательны первые 3 запроса.
