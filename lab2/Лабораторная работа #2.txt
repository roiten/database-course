Задание 1
Последовательно постройте Концептуальную диаграмму по описанной предметной области. База данных должна гарантировать все предъявляемые требования. Задания выполняются в тетради.
1.1* #8
БД для хранения информации о доступных ресурсах гоночного клуба. Проектируемая система должна выполнять следующие действия: 
* Редактировать автопарк - набор гоночных автомобилей, гоночный автомобиль можно добавить, редактировать информацию, отправить в утилизацию. Автомобиль в утилизации более не может катать клиентов.
* Гоночный автомобиль состоит из заголовка, фото, описания, кол-ва лошадиных сил, года выпуска, цвета, наименования подвески.
* Редактировать тренерский состав - добавлять, удалять, редактировать информацию о тренере.
* Информация о тренере включает в себя ФИО, года рождения, номер счёта в банке для начисления заработной платы, фото тренера, список достижений тренера, список автомобилей, на которых тренер может тренировать. А также любимый автомобиль тренера.
* Список достижений тренера состоит из элементов, каждый из которых включает в себя наименование соревнования, дату проведения соревнования и занятое место.
1.2* #10
БД для модуля новостной ленты. Проектируемая система должна выполнять следующие действия: 
* Публиковать новость, состоящую из заголовка, и блоков новости. Блок новости состоит из изображения или абзаца текста. Блоки располагаются в последовательном порядке. Есть возможность. сохранять черновики новостей. В каждой новости есть минимум один блок с изображением.
* Возможность комментировать новость анонимными пользователями, комментарий состоит только из текста. Также можно оставлять ответный комментарий на другие комментарии, а также отвечать на ответы к комментариям. Комментарий можно удалить, но система должна хранить всю историю комментариев, включая ip адрес каждого комментария.
* Просматривать список новостей, который состоит из миниатюры новости, заголовка и кол-ва просмотров новости. Миниатюра извлекается из первого блока с изображением.
* Просмотром новости является запись, состоящая из даты и времени просмотра, а так же ip адреса, с которого просматривали новость.

1.3 #17
Задание разрешается выполнять в редакторе ER диаграммы. БД для бронирования записи на тренировку в гоночный клуб - продолжение задания 1.1. Проектируемая система должна выполнять следующие действия: 
* Хранить информацию о клиенте: ФИО, год рождения, фото (опционально), из какого канала коммуникаций узнал про клуб, номер телефона, уровень езды клиента (новичок, средний уровень, профи, мастер), лидера (опционально). 
* Лидер - клиент, который привёл в систему другого клиента.
* Список каналов коммуникаций клиента редактирует администратор. Канал коммуникаций состоит из наименования, даты начала рекламы, даты конца рекламы. Канал коммуникаций может быть бессрочным, например “сарафанное радио”, а может быть сезонным, например, рекламу по телевизору показывают только летом.
* Календарь бронирования тренировок состоит из возможного времени на тренировку и записей на тренировку. Все бронирования тренировок, гоночного трека, график работы тренера кратны 1 часу.
* Возможное время на тренировку зависит от времени работы клуба (ВС-ЧТ 10:00 - 19:00, ПТ-СБ 12:00 - 23:00), загруженности гоночного трека (гоночный трек иногда бронируют для соревнований), времени работы тренера (указывается индивидуально для каждого тренера), исправности автомобиля, также автомобиль не должен быть забронирован другим клиентом. Также у клуба бывают нерабочие дни.
* Запись на тренировку состоит из клиента, тренера, автомобиля, тренировочных часов (их может быть несколько, но минимум один), пожелания клиента в текстовом формате, состояния записи (забронировано, оплачено, подтверждено, отменено). Тренировочный час состоит из даты и часа начала (от 0 до 23), стоимости тренировки.
* Стоимость тренировки складывается из почасовой стоимости аренды автомобиля, почасовой зарплаты тренера, комиссии гоночного клуба (фиксированная сумма).

1.4 #20
Задание разрешается выполнять в редакторе ER диаграммы. БД для системы бронирования записи на тренировку для гоночных клубов всего мира - продолжение задания 1.3. Проектируемая система должна выполнять следующие действия: 
* Гоночных клубов теперь много. Несколько гоночных клубов могут располагаться на одном треке. Гоночный клуб может располагаться на нескольких гоночных треках. 
* Гоночный трек имеет протяженность, максимальную скорость на треке, сложность (от 1 до 10), адрес, наименование.
* Тренер может работать в разных гоночных клубах и на разных гоночных треках, но только если они расположены в одном городе.
* Автомобиль привязан только к одному гоночному треку.
* При бронировании тренировки клиент теперь может помимо всего прочего выбрать гоночный трек и гоночный клуб. 
* У клиента есть поля профиля для заполнения информации про клиента администратором гоночного клуба или тренером. Список полей профиля редактируется администратором гоночного клуба (у каждого гоночного клуба свой набор полей профиля). Поле профиля может быть текстовым, числовым, обязательным и опциональным, а также списком. Поле типа Список имеет набор возможных значений в порядке приоритета.
